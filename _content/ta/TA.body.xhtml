Grading Abbreviations

Here are some abbreviations I'm starting to use in grading problem sets:

General:

  • ? = DNS = doesn't make sense.
  • DNAQ = doesn't answer the question.
  • IC = incomplete / not enough to get full credit.
  • MM - more mappings (usually for analogical/metaphorical mapping problems)

For PS5:

  • NP - nonsense-parse: your parse tree just doesn't make much sense. Order of symbols mixed up, relationships don't make sense, etc.
  • SPSM - same parse, same meaning: if you have the same parse tree for two sentences with the same function words, the meanings will be basically the same (different actors, same action!)

For mini-esssays:

  • CNC = "consistency in CONCLUSIONS". Lots of people believed that our class definition of "rational" was equivalent to consistent decision-making. No; all that matters is that, given the same evidence, your CONCLUSION is consistent. How you arrive at that conclusion can change however you like.

  • IDT = "ill-defined term" "logic"--what does it mean? No point in "defining" a term in other ill-defined terms!
  • ID = incomplete definition. There's more critical information in the definition of "rational" that you didn't include.
  • PF = proofread; this sentence is not grammatical, is hard to read, or is unreadable
  • WNCD = "why not class definition?" If you didn't use the definition of "rational" from class, I expected you to give some justification for using your own definition. If we define terms in class, why would you make up your own? Would you do that for other terms we learned?
